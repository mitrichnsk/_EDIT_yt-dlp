# ФАЗА ВМ — АССИСТЕНТ РЕЖИССЁРА МОНТАЖА v2.0
### «КОД ДОСТУПА» — ТК «ЗВЕЗДА» | СИСТЕМА ПОИСКА И ВАЛИДАЦИИ ВИДЕОАССЕТОВ

> **Вход:** Утверждённый текст Фазы 4 (подводки) **или** Фазы 5 (ЗК блоков 1–7).
> **Выход:** Реестр валидированных видеоассетов + карта поисковых запросов + шортлист для монтажёра.

---

## ЧТО ИЗМЕНИЛОСЬ В v2.0

В v1.0 фаза ВМ заканчивалась на формировании поисковых запросов.
Редактор получал список ссылок и дальше работал вручную.

В v2.0 фаза ВМ **замыкает петлю**: от текста ЗК до готового файла на диске.
Редактор получает готовые карточки ассетов и только принимает финальное решение: ✓ / — / ✗.

## СТАТУСНАЯ МАШИНА АССЕТА

```
found
  → queued_subtitle
    → subtitle_downloaded
      → pre_scored
        → [score < 0.4] rejected
        → [score ≥ 0.4] queued_visual
          → analyzed_visual
            → approved_for_download   ← решение редактора ✓
            → on_hold                 ← решение редактора —
            → rejected                ← решение редактора ✗
              → downloaded
                → normalized
                  → ready_for_edit
                       │
                       └→ manual_needed  (DRM / geo-block / закрытый источник)
```

## СЛОЙ 1 — QUERY GENERATOR

### ШАГ 1 — ДЕКОМПОЗИЦИЯ БЛОКА ЗК НА СУБ-БЛОКИ

Перед формированием поисковых запросов ОБЯЗАТЕЛЬНО:

1. Прочитай весь текст блока ЗК целиком.
2. Выдели все смысловые «кадровые единицы» — персонажи, места,
   объекты, действия, документы, символы, эмоции.
3. Раздели блок на МИНИМУМ 7 суб-блоков. Каждый суб-блок =
   один отдельный визуальный акцент в монтаже.
4. Суб-блоки делятся на два типа:

   ● БУКВАЛЬНЫЙ — прямое изображение того, о чём говорится в ЗК
     Пример ЗК: «...Kawasaki создала фонд в 1 миллиард иен...»
     Суб-блок: Завод Kawasaki в Кобе / Документы / Адмиралы в форме

   ● КОНТЕКСТНЫЙ — атмосферный, смысловой или метафорический план,
     усиливающий восприятие сказанного без буквального совпадения
     Пример ЗК: «...Кэйданрэн управляет страной как дзайбацу...»
     Суб-блок: Деловой квартал Токио с дрона / Тени людей в переговорной /
               Мемориал дзайбацу архивное фото

5. **НОВОЕ В v2.0:** Каждому суб-блоку присвоить тег `mode`:

   - `shot_search` — нужен конкретный кадр конкретного события/человека/места
   - `quote_search` — нужна цитата, выступление, интервью конкретного человека
   - `archive_search` — нужна хроника, исторические кадры, архивный материал
   - `broll_search` — атмосферный B-roll, без привязки к конкретному событию

### ШАГ 2 — ПОИСК ПО 5 ПЛАТФОРМАМ

Для каждого суб-блока формировать запросы на ВСЕХ ПЯТИ платформах:

1. 📺 YOUTUBE
   Видеозаписи: новостные сюжеты, хроника, документальные фрагменты,
   пресс-конференции, съёмки с мероприятий.

2. 🖼️ GOOGLE PHOTO (Google Images)
   Фотографии: архивные, новостные, официальные, репортажные.

3. 🎬 GOOGLE VIDEO
   Видео общего поиска Google: репортажи, интервью, прямые трансляции,
   которые могут не попасть в YouTube.

4. 📰 ТОПОВЫЕ СМИ (Reuters, Bloomberg, AP, AFP и т.д.)

5. 📡 РЕГИОНАЛЬНЫЕ СМИ
   Подбираются по теме выпуска. Примеры:
   - Япония: NHK, Kyodo News, Nikkei Asia, The Japan Times
   - США: Politico, The Hill, ProPublica
   - Европа: Der Spiegel, Le Monde, Guardian

### ШАГ 3 — ПРАВИЛА ФОРМИРОВАНИЯ ЗАПРОСОВ

ПРАВИЛО 1 — ДВУЯЗЫЧНОСТЬ (ОБЯЗАТЕЛЬНО):
Каждый поисковый запрос пишется на русском и английском.
Формат: [Запрос на русском] / [Search query in English](URL)

✅ ПРАВИЛЬНО:
[Остров Йонагуни радар военная база] / [Yonaguni island radar military base Japan](URL)

❌ НЕПРАВИЛЬНО:
[Japan military base](URL) — нет русского / нет конкретики

---

ПРАВИЛО 2 — МАКСИМАЛЬНАЯ СПЕЦИФИКАЦИЯ:
Чем точнее запрос — тем ценнее результат.

✅ ХОРОШО:   [Kawasaki Heavy Industries завод подводные лодки Кобе]
✅ ЛУЧШЕ:    [Kawasaki Heavy Industries Kobe submarine factory aerial 2024]
❌ ПЛОХО:    [завод / factory]

---

ПРАВИЛО 3 — КОНТЕКСТНЫЕ ЗАПРОСЫ:
На каждый буквальный запрос должен быть минимум 1 контекстный.

Пример:
● Буквальный:  [Деньги переданы в конверте взятка архив] / [money envelope bribe archive footage]
● Контекстный: [Теневые переговоры силуэты за стеклом] / [silhouettes shadow meeting glass door]

---

ПРАВИЛО 4 — ЗАПРЕТ НА АБСТРАКЦИЮ:
Если в ЗК упомянут конкретный человек, объект или место —
искать именно его, а не общий образ.

❌ ЗАПРЕЩЕНО: [японский политик / Japanese politician]
✅ ПРАВИЛЬНО: [Санаэ Такаити пресс-конференция] / [Sanae Takaichi press conference 2026]

---

ПРАВИЛО 5 — ПРИОРИТЕТ АУТЕНТИЧНОСТИ:
1. Приоритет: Реальные архивные кадры, прямые трансляции, репортажи
2. Второй уровень: Официальные фото, документы, инфографика
3. Третий уровень: Контекстные кадры (метафора, атмосфера)
4. Только если нет реального материала: инфографика, 3D-анимация

---

ПРАВИЛО 6 — НОВОЕ В v2.0 — YOUTUBE-ПЕРВИЧНОСТЬ:
YouTube-запросы имеют приоритет, так как именно они уходят в
автоматический Pipeline (Слои 2–5). Для каждого суб-блока формировать
МИНИМУМ 3 YouTube-запроса — они будут обработаны автоматически.

## СЛОЙ 2–3 — АВТОМАТИЧЕСКИЙ PIPELINE (yt-dlp)

После сохранения `vm_search_map.json` агент запускает автоматическую
обработку YouTube-запросов через скрипт `vm_pipeline.py`.

### Команды yt-dlp для Слоя 3 (субтитры без скачивания)

```bash
# Получить субтитры без скачивания видео:
yt-dlp --skip-download \
       --write-auto-sub \
       --write-sub \
       --sub-lang ru,en \
       --sub-format srt \
       --convert-subs srt \
       --output "%(id)s.%(ext)s" \
       "https://www.youtube.com/watch?v=VIDEO_ID"

# Получить только метаданные:
yt-dlp --skip-download \
       --print-json \
       "https://www.youtube.com/watch?v=VIDEO_ID"

# Полное скачивание одобренного ассета (Слой 5):
yt-dlp --format "bestvideo[height<=1080][ext=mp4]+bestaudio[ext=m4a]/best[height<=1080]" \
       --merge-output-format mp4 \
       --write-info-json \
       --write-thumbnail \
       --add-metadata \
       --output "{scene_id}_{source}_{id}.%(ext)s" \
       "URL"
```

### Pre-screening промпт для Слоя 3 (LLM-оценка транскрипта)

```
SYSTEM: Ты — ассистент режиссёра монтажа. Оценивай транскрипты видео
на предмет пригодности для использования в качестве видеоряда (B-roll
или синхрон) в аналитической программе.

USER:
ЗАДАЧА МОНТАЖА: {editorial_need}
ЗК-ФРАГМЕНТ: {zk_fragment}
MODE: {mode}

ТРАНСКРИПТ ВИДЕО:
{transcript_text}

ОЦЕНИ по шкале 0.0–1.0:
- fit_score: насколько содержание видео соответствует задаче монтажа
- has_speaker: есть ли узнаваемый спикер (для quote_search)
- usable_as_broll: можно ли использовать как B-roll без звука
- language_match: соответствует ли язык запросу

Верни JSON:
{
  "pre_fit_score": 0.0–1.0,
  "has_speaker": true/false,
  "usable_as_broll": true/false,
  "language_match": true/false,
  "relevant_timestamps": ["00:12–00:45", "02:10–02:30"],
  "reject_reason": "строка или null"
}
```

## СЛОЙ 5 — ИНТЕРФЕЙС ВАЛИДАЦИИ ДЛЯ РЕДАКТОРА

После автоматической обработки агент выводит **шортлист карточек**.
Редактор принимает решение по каждой.

### Формат карточки для редактора

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎬 АССЕТ [asset_id]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 Суб-блок:    B1.1 — [название]
📋 Задача:      [editorial_need]
🔗 Источник:    [channel] | [publish_date] | [duration]
🔗 URL:         [source_url]

🤖 AI-оценка:
   pre_fit_score:   0.82 ████████░░
   visual_fit:      0.88 █████████░
   Под голос:       ✅ да
   Спикер:          ✅ есть

⏱️ Ключевые сегменты:
   00:12–00:45 → «Министр выходит из здания, машет рукой»
   02:10–02:30 → «Крупный план документа на столе»

📝 Редакционная заметка: «Хороший план для перебивки, чистый фон»

🔒 Права: YouTube / стандартная лицензия

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Решение: [✓ СКАЧАТЬ]  [— РЕЗЕРВ]  [✗ ОТКЛОНИТЬ]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

После решения редактора:
- `✓ СКАЧАТЬ` → yt-dlp полный download → статус `downloaded` → `ready_for_edit`
- `— РЕЗЕРВ` → статус `on_hold`, не скачивается
- `✗ ОТКЛОНИТЬ` → статус `rejected`

## ЧЕК-ЛИСТ ПЕРЕД СДАЧЕЙ v2.0

### Качество запросов (наследовано из v1.0):
- [ ] В каждом блоке ЗК не менее 7 суб-блоков?
- [ ] Среди суб-блоков есть оба типа (БУКВАЛЬНЫЙ + КОНТЕКСТНЫЙ)?
- [ ] Каждый запрос написан на русском И английском?
- [ ] Использованы все 5 платформ поиска?
- [ ] Нет абстрактных запросов (если есть конкретика)?
- [ ] Все ссылки ведут на правильные URL-адреса поиска?
- [ ] Для каждого суб-блока минимум 3 YouTube-запроса?
- [ ] Каждому суб-блоку присвоен mode-тег?
- [ ] Поле editorial_need заполнено для каждого суб-блока?

### Автоматический pipeline (новое в v2.0):
- [ ] vm_search_map.json сохранён и валиден?
- [ ] YouTube-кандидаты прошли pre-screening (субтитры)?
- [ ] Кандидаты с pre_fit_score < 0.4 автоматически отклонены?
- [ ] Для топ-5 кандидатов запущен Gemini Visual Analyzer?
- [ ] Карточки ассетов сформированы с полями gemini_analysis?
- [ ] Редактор получил шортлист карточек (не ссылок)?
- [ ] Система не скачала ни одного файла без approved-статуса?

### Выходные артефакты:
- [ ] vm_asset_registry.json сохранён?
